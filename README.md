# LIO-RTK-Orchard-Navigation

This repository contains the system framework, configuration files, evaluation scripts, and semantic map outputs for the paper focusing on **robust LIO-RTK localization, semantic mapping, and autonomous navigation for tracked UGVs in non-standardized orchards**.

---

## 1. Core Source Code Release Policy

To protect intellectual property (IP) prior to publication, the underlying C++ implementations of the core algorithmic modules—specifically the asynchronous ESIKF fusion node and the sub-pixel tree instance extraction node—have been temporarily removed from this branch. 

**The full, executable ROS workspace and complete source code will be officially open-sourced to the community upon the acceptance of this paper.**

---

## 2. Repository Structure & Reproducibility

To assist reviewers in verifying the authenticity of our real-world experiments and the reproducibility of our quantitative metrics, we provide the complete engineering skeleton, hardware interfaces, and evaluation tools. The repository is organized as follows:

### 📍 `/fast_lio_navigation`
Contains the authentic physical parameters and tuned planners used in our real-world tracked UGV autonomous navigation experiments (Validating **Section 3.4** of the paper).
* **`/param`**: Highly customized configurations for undulating orchard terrains, including `teb_local_planner_params.yaml`, `costmap_common_params.yaml`, and `global_planner_params.yaml`.
* **`/maps`**: Pre-built 2D safety costmaps (`orchard_map.pgm` & `.yaml`) generated by our semantic mapping module.

### 📍 `/rtk_driver/scrpic` (Evaluation Scripts)
Contains the core Python scripts used to process the datasets and generate the evaluation results and figures presented in the paper.
* `extract_rtk_gt.py`: Extracts high-precision RTK ground truth with lever-arm extrinsic compensation.
* `simulate_signal_outage.py`: Explains how the degraded GNSS dataset was controlled and generated.
* `eval_naive_baseline.py`: Reproduces the "Naive RTK-LIO" baseline comparison shown in **Table 2**.
* `plot_signal_degradation.py`: Generates the spatiotemporal analysis heatmaps in **Figure 7**.
* `eval_nav_smoothness.py`: Evaluates navigation accuracy and chassis smoothness metrics shown in **Table 7**.

### 📍 `/orchard_semantic_map`
* `tree_inventory.csv`: The final extracted semantic map inventory (WGS84 coordinates and instance IDs).
* `map.rviz`: RViz visualization configurations for displaying the semantic output.

### 📍 `/LIO-RTK` & Hardware Drivers
Contains the system headers, state-estimation mathematical dependencies, and calibration files.
* **`/LIO-RTK/config`**: Actual sensor calibration files (`velodyne.yaml`, `localization.yaml`).
* **`/LIO-RTK/include`**: Core mathematical tools including `ikd-Tree` and `IKFoM_toolkit` used by the front-end LIO.
* **Hardware Drivers**: Authentic drivers for the 3D LiDAR (`/lslidar_driver`), RTK receiver (`/rtk_driver`), and 10-axis IMU (`/wit_imu_driver`) deployed on our Intel NUC edge computing platform.
